# 四平台抓取策略（分层降级）

## 来源优先级
- 首选一手、高质量来源：官方文档、项目博客、标准/规范、原始仓库、作者本人 RSS/Atom feed。
- 次选高信号整理来源：长期可信的技术博客、研究团队更新、维护良好的社区索引。
- 谨慎使用二手聚合、短帖和平台转发；除非它们指向原始材料，否则优先级应低于一手来源。
- 本地 MVP 通过 `priority_domains` 和 `rss_feeds[].priority` 表达这些偏好，并在规范化 URL 后统一去重。

## YouTube
- 首选：字幕 API
- 失败：`yt-dlp + whisper`
- 若再失败：要求用户补音频/视频文件

## 小红书
- 先抓：标题 + 描述
- 常见缺失：正文
- 补料：正文粘贴或截图

## 公众号
- 先尝试：`wemp.app` 镜像
- 常见缺失：全文
- 补料：正文粘贴（最稳）

## X/Twitter
- 先抓：oEmbed 正文
- 视频内容：建议补充描述或本地转录
